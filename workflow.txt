============================================================
Campus Placement Tracker System — Milestone 1 Workflows
============================================================

Workflow 1: Student Applies to a Job (Eligibility Validation)
Actor(s): Student (primary), System
Goal: Student successfully applies only if eligible.

Preconditions:
- Student is logged in
- Student profile contains academic details
- Job exists and deadline not passed

Main Steps:
1) Student logs in
2) Student opens Job Listings
3) System displays available jobs
4) Student opens job details
5) Student clicks Apply
6) System validates eligibility (CGPA, program, backlogs, deadline)
7) If valid, system creates Application with status = Applied
8) System confirms submission
9) Student views status in My Applications

Alternate / Error Flows:
- Not eligible → System blocks application
- Deadline passed → System blocks application
- Duplicate application → System prevents double apply

Postconditions:
- Application stored if eligible
- Student can track status

------------------------------------------------------------

Workflow 2: Recruiter Creates a Job Posting (Eligibility + Deadline)
Actor(s): Recruiter (primary), System
Goal: Recruiter posts a job with criteria and deadline.

Preconditions:
- Recruiter is logged in

Main Steps:
1) Recruiter logs in
2) Recruiter opens Dashboard
3) Recruiter clicks Create Job
4) Recruiter enters job details
5) Recruiter sets eligibility criteria
6) Recruiter sets deadline
7) Recruiter clicks Publish
8) System validates inputs
9) System saves job posting

Alternate / Error Flows:
- Missing fields → Validation error
- Invalid deadline → Block publish

Postconditions:
- Job visible to students

------------------------------------------------------------
Workflow 3: Interview Scheduling & Slot Booking (Conflict Detection)
Actor(s): Recruiter, Student, System
Goal: Schedule interviews without conflicts.

Preconditions:
- Student shortlisted
- Recruiter logged in

Main Steps:
1) Recruiter selects candidate
2) Recruiter creates slots
3) System publishes slots
4) Student selects slot
5) System checks conflicts
6) System confirms booking
7) System updates status

Alternate / Error Flows:
- Slot taken → Ask student to choose another
- Conflict detected → Block booking

Postconditions:
- Interview scheduled
